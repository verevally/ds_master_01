Описание проекта:
Предсказание (преждевременного) закрытия финансируемых стартапов
Для проекта предоставлены псевдо-реальные данные (реальные данные, в которые добавлена синтетическая составляющая) о стартапах, функционировавших в период с 1970 по 2018 годы.
Задача предсказать, какие из представленных стартапов закроются.
Срок проведения соревнования на платформе Kaggle составляет 2 недели.
С 03.06.2024 по 17.06.2024
www.kaggle.com/competitions/startups-operations-close-predictions-m1-31ds

Цель исследования:
Разработать решение, которое позволит предсказать преждевременное закрытие стартапа.

Задачи исследования:
01. загрузка и ознакомление с данными,
02. предварительная обработка,
03. полноценный разведочный анализ,
04. разработка новых синтетических признаков,
05. проверка на мультиколлинеарность,
06. отбор финального набора обучающих признаков,
07. выбор и обучение моделей,
08. итоговая оценка качества предсказания лучшей модели,
09. анализ важности её признаков,
10. подготовка отчёта по исследованию;

+ Дополнительные задания:
11. реализовать решение с использованием технологии pipeline (из библиотеки sklearn / imblearn)
12. выполнить полноценный исследовательский анализ - выделить важные компоненты из данных, влияющие на работу стартапов, 
    на основе данных и графиков сформулировать рекомендации, позволяющие повысить шанс на успех стартапа
13. Подготовка полноценного отчёта по исследовательской работе Дата Сайнтиста. Примеры Stanford University:
https://cs229.stanford.edu/projects2014.html


Используемые модели МО:
1. Метод k-ближайших соседей;

3. Метод опорных векторов;
4. Логистическая регрессия / линейная классификация (выключена из опубликованного решения).

Статус к финалу соревнования:
-01. не применены новые библиотеки и методы, не описанные в основном курсе (imblearn, catboost, lightgbm, phik)
-02. полноценный резведочный анализ входящих данных не выполнен на желаемом уровне
-03. корреляционный анализ выполнен без использования библиотеки phik, - выбранный метод Спирмена может не учитывать особенности в данных
-04. в финальном наборе факторов присутствует сомнительный фактор lifetime - являющийся утечкой целевого признака
-05. анализ важности признаков лучшей модели указывает на преобладающий вес фактора lifetime, являющегося утечкой целевого признака
-06. отчёт по исследованию не выполнен на желаемом уровне
-07. не выполнен полноценный исследовательский анализ: не выделены важные компоненты из данных, влияющие на работу стартапов и не сформулированы рекомендации, позволяющие повысить шанс на успех стартапа
-08. не подготовлен полноценный отчёа по исследовательской работе Дата Сайнтиста в соответствии с рекоммендациями
+09. использован sklearn.pipeline.Pipeline, однако без формирования единого, объединяющего предобработку и обучение модели, так как отдельный рефакторинг синтетических факторов не вписывается в пайплайн
+10. разработаны интересные (однако, не факт что полезные) синтетические факторы: 
	- десятичный логарифм общей суммы финансирования
	- средняя сумма инвестиций на 1 раунд, количество раундов более одного
	- доткомы, инкорпорэйтед из названия
	- unstack категорий, частотность категорий
+11. после рассчёта проб (вероятностей отнесения наблюдения к целевому признаку), рассчитан (на тестовой выборке) ПОРОГ threshold отнесения к целевому классу
+12. с учётом ПОРОГА, финальное предсказание показало целевую метрику 100%
+13. попытка рассмотреть задачу не как бинарную классификацию, а как линейную регрессию продолжительности жизни стартапа и последующим сравнением с датой формирования отчёта
